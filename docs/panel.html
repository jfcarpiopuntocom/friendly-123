Roles y aclaraciones

🕵️ Master (tú, JFC)

Controlas versiones, parches, códigos maestros y despliegues de todas las apps. Este panel es tuyo. Accedes con tu PIN de 3 dígitos.

🏪 Dueño / Admin de local

Opera su negocio con sus propios PINs (dueño, empleado, contable). Ej: Jose es el dueño de Olimpo Control. NO es master. No ve este panel.

⚠️ No confundir productos

Olimpo Control (POS/inventario de Jose, smoke shop en Cuenca) es un producto totalmente distinto de Olimpo 2.0 (consola de música/chat del club, auth por Nostr). Repos, dominios y bases de código separados. Nunca tratar uno como copia del otro.

Notas operativas

Liberar correo — ¿qué hace exactamente?
Borra el campo email dentro de localStorage["oc_secure"] del cliente (misma origin github.io). El dueño vuelve a ver el formulario de correo en su sección Avanzado, sin tocar sus PINs ni el resto de su configuración.

Atención: esto afecta la demo en esta misma pestaña. Para producción (servidor propio), ir físicamente al dispositivo del cliente o usar la entrada de código maestro en Avanzado.
Códigos maestros — cómo se validan
En Olimpo Control, verificarMaestro() revisa siempre la lista MASTER_CODES_AUTHORIZED, aunque el navegador ya tenga un hash guardado. Por eso basta refrescar para que un código nuevo funcione — el dueño no tiene que borrar nada.
Activación Loyverse (Olimpo Control)
Jose obtiene su token en Loyverse → app → Avanzado → "Conectar Loyverse" → guarda → reinicia node server.js.
EmailJS — recuperación de acceso
Configurar EMAILJS_CONFIG en email-recovery.js de cada proyecto. Hasta entonces, el código de reset aparece en pantalla.
Mantener este panel al día
Los números de versión/commit son un snapshot manual (una página estática no consulta git en vivo). Al pushear cambios grandes, actualiza las cards y la tabla de arriba con el nuevo hash y fecha.
